# P0.3 — SƠ ĐỒ LẮP MẠCH THỰC TẾ (bố trí nút + breadboard)

> Bổ sung cho `P0.2_PINMAP_CubeMX.md` (đã chốt pin-map). File này tập trung vào
> **bố trí vật lý**: đặt nút ở đâu cho thuận tay, đấu breadboard thế nào cho gọn,
> tránh nhầm dây khi ráp thật.

---

## 1) Tổng quan bố trí trên bàn

Hai người chơi ngồi đối diện board, mỗi người 1 cụm điều khiển riêng + breadboard riêng
(để không giẫm dây nhau). Buzzer đặt gần board (dùng chung, không thuộc về ai).

```
                    ┌───────────────────────────────────┐
                    │   STM32F429I-DISC1 (đứng giữa)     │
                    │        ┌──────────────────┐        │
                    │        │   LCD 240x320    │        │
                    │        └──────────────────┘        │
                    │   [B1]=PAUSE      LD3🟢 LD4🔴       │
    HEADER P1 ──────┤                                    ├────── HEADER P2
   (trái board)     │                                    │      (phải board)
                    └───────────────────────────────────┘
                          │                    │
                     dây nối                dây nối
                          │                    │
              ┌───────────▼──────────┐  ┌──────▼───────────────┐
              │   BREADBOARD P1      │  │    BREADBOARD P2      │
              │  (người chơi nút)    │  │  (người chơi joystick) │
              │  7 nút D-pad style   │  │  joystick + 2 nút      │
              └──────────────────────┘  └────────────────────────┘
                                              (buzzer cắm cạnh board,
                                               dây tới PC8 trên header P1)
```

> Nếu chơi **1 người** (không PvP): chỉ cần lắp breadboard P1 (7 nút). Joystick +
> 2 nút PB7/PC12 để dành khi ráp PvP.

---

## 2) Bố trí nút BREADBOARD P1 (người chơi nút — 7 nút kiểu tay cầm)

Xếp theo layout **arcade D-pad**: khối xoay ở trên/dưới, trái/phải hai bên,
3 nút hành động xếp hàng dưới cùng — bấm bằng ngón cái không bị chéo tay.

```
                     ┌────────────┐
                     │  ROT_CCW   │   PD4   (xoay trái)
                     └────────────┘
        ┌────────┐                  ┌────────┐
        │  LEFT  │                  │ RIGHT  │   PG2 / PG3
        └────────┘                  └────────┘
                     ┌────────────┐
                     │  ROT_CW    │   PC11  (xoay phải)
                     └────────────┘

   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐
   │ SOFT_DROP   │   │ HARD_DROP   │   │  HOLD (P1)  │
   │   PD2       │   │   PD7       │   │    PD5      │
   └─────────────┘   └─────────────┘   └─────────────┘
```

Mỗi nút đấu **1 chân GPIO ─ nút ─ GND** (breadboard có sẵn rail GND chạy dọc,
đấu tất cả 7 nút + PAUSE dùng chung 1 rail GND, rồi 1 dây duy nhất từ rail đó
ra chân GND của board).

---

## 3) Bố trí BREADBOARD P2 (người chơi joystick — chỉ cần khi chơi PvP)

```
                    ┌───────────────────────┐
                    │       JOYSTICK        │
                    │   (module KY-023)     │
                    │                       │
                    │  VRx → PC3  (trái/phải)│
                    │  VRy → PA7  (không dùng*)│
                    │  SW  → PG9  (nhấn = xoay CW)│
                    │  VCC → 3V3, GND → GND │
                    └───────────────────────┘

        ┌───────────────┐        ┌───────────────┐
        │  CCW (P2)     │        │  HOLD (P2)     │
        │    PC12       │        │    PB7         │
        └───────────────┘        └───────────────┘
           (2 nút đặt              (2 tay cái bấm
            2 bên joystick,         dễ với khi tay
            dễ với ngón cái)        kia giữ cần)
```

`*` VRy hiện chưa được engine dùng để rơi nhanh (chỉ VRx quét trái/phải + SW
xoay) — vẫn đấu dây đủ 4 chân theo pin-map để tránh phải tháo lắp lại nếu
sau này bổ sung tính năng.

---

## 4) Buzzer — vị trí trung lập

Buzzer không thuộc về người chơi nào, nên cắm ngay cạnh board (breadboard nhỏ
riêng hoặc chung với board), dây ngắn nhất có thể để giảm nhiễu PWM:

Module có **3 chân** trên header, nhưng chỉ 2 chân có tác dụng — chân giữa
(ghi "VCC"/"5V" cho giống các module KY khác) **không nối gì cả** vì loa áp
điện bên trong chỉ có 2 dây thật:

```
   KY-006 (passive buzzer) — nhìn từ trên xuống
   ┌───────────────────────┐
   │  S      middle    -   │
   │  ●        ●       ●   │
   └──┼────────┼───────┼───┘
      │        │       │
      ▼        ▼       ▼
     PC8    (bỏ trống  GND
  (TIM3_CH3   không
   header P1)  nối)
```

## 5) Bảng màu dây khuyến nghị (tránh cắm nhầm khi có 13 dây)

| Màu dây | Dùng cho |
|---|---|
| **Đỏ** | 3V3 (chỉ joystick VCC) |
| **Đen** | GND (tất cả nút + joystick GND + buzzer -) |
| **Vàng** | Tín hiệu ADC joystick (VRx, VRy) |
| **Xanh dương** | Tín hiệu buzzer (PC8) |
| **Trắng** | 7 nút Player 1 (LEFT/RIGHT/CW/CCW/SOFT/HARD/HOLD) |
| **Cam** | 2 nút Player 2 (HOLD2, CCW2) + joystick SW |

Quy tắc: **1 màu = 1 loại tín hiệu**, không lẫn giữa GND và tín hiệu — nhìn màu
dây là biết ngay đang đấu cái gì, đỡ phải dò lại từng chân khi debug.

---

## 6) Thứ tự lắp ráp (test từng cụm trước khi lắp tiếp)

1. **GND chung trước** — 1 dây đen từ board ra rail GND của cả 2 breadboard.
2. **Buzzer** — cắm PC8 + GND, nạp thử `buzzer_tone(1000)` nghe có kêu không.
3. **7 nút Player 1** — cắm từng nút, nạp firmware test đọc `HAL_GPIO_ReadPin`,
   xác nhận nhấn xuống mức 0.
4. **PAUSE (B1 on-board)** — không cần đi dây, có sẵn.
5. **Joystick** — cắm 3V3 (không 5V!), GND, VRx, VRy, SW; test đọc ADC ra
   0–4095 khi gạt cần.
6. **2 nút Player 2 (PvP)** — HOLD2 (PB7), CCW2 (PC12); chỉ cần khi test PvP.
7. **LED trạng thái âm thanh** — có sẵn trên board (PG13/PG14), không đi dây.

Test xong cụm nào thì mới lắp cụm tiếp theo — lỗi ở đâu khoanh vùng được ngay,
không phải tháo hết ra dò lại.

---

## 7) Checklist trước khi đóng nắp breadboard

- [ ] Không có nút/joystick nào đấu vào **chân bẫy** (xem `P0.2_PINMAP_CubeMX.md`
      mục "Bài học quan trọng").
- [ ] Joystick chắc chắn cắm **3V3**, không phải 5V.
- [ ] Rail GND của 2 breadboard nối chung về **1 điểm GND** trên board (tránh
      ground loop nếu chỉ cắm rời từng breadboard vào board qua nhiều điểm GND
      khác nhau — vẫn được nhưng nên gom 1 điểm cho gọn).
- [ ] Buzzer là loại **thụ động** (test bằng cách đổi tần số phải nghe đổi tông,
      không phải kêu 1 tông cố định).
- [ ] Dây tín hiệu ngắn nhất có thể quanh khu vực joystick/buzzer (giảm nhiễu ADC/PWM).
